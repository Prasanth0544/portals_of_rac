
Here's your interview story — a clear narrative flow for when a recruiter asks "How did you do it?":

🎤 The Interview Answer (2-minute version):
"My RAC project needed train route data for 12,000+ trains. I started by scraping erail.in — but that took hours with Selenium. So I looked for a faster way."

Step 1 — I found the data source:

"The 'Where is My Train' app works offline, so I knew the data must be embedded inside the APK. I pulled the APK from my phone using ADB, extracted it with 7-Zip, and found 6 SQLite databases and several binary files with obfuscated names like 

ladoo
, 

boondi
, 

munthiri
 — the dev team used food names to hide file purposes."

Step 2 — I traced the code:

"I loaded the APK into JADX (a Java decompiler) which decompiled 14,000+ classes. I searched for 

DataInputStream
 and getAssets().open to find which class reads which file. This led me to class lxh — the main database loader. The key method G() had 723 bytecode instructions that JADX initially refused to decompile, so I switched to debug mode to force it."

Step 3 — I understood the binary format:

"By reading the decompiled Java, I mapped out the exact byte layout — every train record starts with a 4-byte Big Endian integer for train number, followed by packed dates, departure time, and then 14 bytes per station (GID, time delta, distance, halt duration). Station names aren't stored as strings — they use 2-byte integer GIDs that map to station codes through a SQLite lookup table. This saves megabytes on a mobile app."

Step 4 — I wrote the decoder:

"I translated the Java parsing logic into a Python script using the struct module for Big Endian unpacking and sqlite3 for the GID→station code mapping. The script reads the 4.6 MB binary sequentially, decodes each train record, and saves to MongoDB."

Step 5 — Result:

"10,456 train routes decoded in 5 seconds — replacing a Selenium scraper that took hours. I verified the output against erail.in — station codes, departure times, distances all matched. The data now powers my RAC system's route-based features."
